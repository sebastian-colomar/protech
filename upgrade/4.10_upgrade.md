## Update Graph

In the following link you can check the Certified Cloud Pak for Data System upgrades:
- https://www.ibm.com/docs/en/cloud-paks/cloudpak-data-system/2.0.0?topic=upgrading-cloud-pak-data-system

You current status should be:
- Cloud Pak for Data System: 2.0.2.1
- Red Hat® OpenShift® Container Platform:	4.8.37
- Red Hat Enterprise Linux®:	8.6
- OpenShift Data Foundation:	4.8.7
- Cloud Pak for Data:	4.0.2
- Netezza® Performance Server:	11.2.2.4
- Cumulus:	3.7.11
- Python:	3.6.8

Check the Red Hat OpenShift Container Platform Update Graph:
- https://access.redhat.com/labs/ocpupgradegraph/update_path/

This link shows the update graph from version 4.8.37 to 4.10.64:
- https://access.redhat.com/labs/ocpupgradegraph/update_path/?channel=stable-4.8&arch=x86_64&is_show_hot_fix=false&current_ocp_version=4.8.37&target_ocp_version=4.10.64

These are the steps described in the previous link as retrieved at the date of composing this guide:
  - To Select the stable-4.9 channel, run this patch command on the CLI:
    ```
    oc patch clusterversion version --type merge -p '{"spec": {"channel": "stable-4.9"}}'
    ```
  - Upgrade the cluster from 4.8.37 to 4.9.59.
  - Select the stable-4.10 channel:
    ```
    oc patch clusterversion version --type merge -p '{"spec": {"channel": "stable-4.10"}}'
    ```
  - Upgrade the cluster from 4.9.59 to 4.10.64.
